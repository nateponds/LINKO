# Example Backlog

Use this example when a teammate wants to suggest a task, feature, improvement, fix, research item, or technical decision that the whole team should see.

Backlog items are not automatically active work. They should be reviewed, clarified, prioritized, and moved into `SPRINTS.md` only when the team agrees to work on them.

## Backlog Item Template

### [Short Task or Idea Title]

Status: Proposed  
Suggested by: @github-username  
Date added: YYYY-MM-DD  
Area: Frontend / Backend / DevOps / Design / Product / Documentation  
Priority: Low / Medium / High  
Related docs: ROADMAP.md / FRONTEND_GUIDE.md / BACKEND_GUIDE.md / None

Description:
Briefly explain the task, idea, problem, or improvement.

Reason:
Why should the team consider this?

Expected outcome:
What should be true if this is eventually completed?

Notes:
Any extra context, links, screenshots, questions, or concerns.
