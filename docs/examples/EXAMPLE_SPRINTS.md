# Example Sprints

Use this example when a high-authority teammate approves work, or when the team agrees to start work on selected tasks.

Sprint items should usually come from `BACKLOG.md`, `ROADMAP.md`, or an approved team discussion. A sprint should describe what the team is committing to work on now.

## Sprint Template

### Sprint Name

Date range: YYYY-MM-DD to YYYY-MM-DD  
Sprint lead: @github-username  
Status: Planned / Active / Completed

Goal:
Briefly describe what this sprint is meant to accomplish.

Approved by:

- @github-username
- Team agreement

Tasks:

- [ ] Task name
  - Owner: @github-username
  - Area: Frontend / Backend / DevOps / Design / Product / Documentation
  - Source: BACKLOG.md / ROADMAP.md / Team discussion
  - Notes: Short implementation note

Blockers:

- None yet

Review notes:
Add summary after the sprint ends.
